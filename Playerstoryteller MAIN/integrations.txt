## EXECUTIVE SUMMARY
================================================================================

**Mission**: Transform Ratlab from a simple viewer events system into a comprehensive
interactive streaming platform with economy, voting, and direct colonist control.

**Core Philosophy**:
- Platform-agnostic (works beyond Twitch)
- Optional Twitch OAuth for enhanced features
- Maintain anonymity as default (no forced karma tracking)
- Dual-mode system: "Event Mode" (current) + "Control Mode" (new)

**Key Architectural Change**:
MIGRATE FROM IN-GAME MOD SETTINGS TO WEB-BASED SETTINGS DASHBOARD

**Priority**: CRITICAL - Do this FIRST before any other features
**Estimated Time**: 1-2 weeks

### Why This Matters:
- Current mod settings are clunky and dont look good.

- Foundation for future admin features

### Implementation:

**Features**:
- Session management (public/private toggle, password)
- Action enable/disable toggles (replace mod settings page)
- Economy settings (coin rates, action costs)
- Rate limiting configuration
- Queue settings (voting duration, approval mode)
- Colonist adoption settings (cost, max adoptions)
- Live viewer list with coin balances
- Action history/log viewer
- Real-time session statistics

#### 2. Settings API Endpoints
```
GET  /api/settings/:sessionId          - Get all settings
POST /api/settings/:sessionId          - Update settings
POST /api/settings/:sessionId/validate - Validate auth token
```

#### 3. Settings Sync System
- Server stores settings in session object
- Mod polls `/api/settings/:sessionId` every 5 seconds
- Apply settings changes without game restart
- Settings persist in database/file storage

#### 4. Authentication for Dashboard
- Streamer authenticates with session ID + secret key
- Optional: Twitch OAuth for streamers
- Settings changes require authentication
- Dashboard access separate from viewer access

### Migration Strategy:
1. Keep existing mod settings as fallback/defaults
2. Web settings override mod settings when available
3. Gradual migration - move one setting category at a time
4. Test with both systems running in parallel

```csharp
[Route("POST", "/api/v1/colonist/command")]
public void HandleColonistCommand(HttpListenerContext context)
{
    var body = ReadBody(context);
    var command = JsonConvert.DeserializeObject<ColonistCommand>(body);

    // Validate ownership via ViewerManager
    if (!ViewerManager.CanControlColonist(command.username, command.colonistId))

public class ColonistCommand
{

public static void ExecuteColonistCommand(ColonistCommand cmd)
{
    Pawn pawn = Find.CurrentMap.mapPawns.AllPawns

        case "move":
            Job job = JobMaker.MakeJob(JobDefOf.Goto, (IntVec3)cmd.target);
            pawn.jobs.TryTakeOrderedJob(job);
            break;

        case "attack":
            // ... attack logic
            break;

        // ... other commands
    }
}
```

### 5.1: Map Capture System

**Option A: GPU Async Screenshot (Your Suggestion)**
Use existing BitMiracle.LibJpeg.NET with GPU acceleration:

```csharp
// MapCaptureManager.cs
public class MapCaptureManager
{
    public byte[] CaptureFullMap()
    {
        // Switch camera to top-down orthographic
        Camera mapCam = CreateMapCamera();

        // Render full map to RenderTexture
        RenderTexture rt = RenderFullMap(mapCam);

        // Async GPU readback
        AsyncGPUReadback.Request(rt, 0, TextureFormat.RGB24,
            OnMapCaptureComplete);
    }

    private void OnMapCaptureComplete(AsyncGPUReadbackRequest request)
    {
        if (request.hasError) return;

        // Convert to JPEG using LibJpeg.NET
        byte[] jpegData = CompressToJpeg(request.GetData<byte>());

        // Send to server
        SendMapUpdate(jpegData);
    }
}
```

**Settings**:
- Capture frequency: 500ms-1000ms (configurable)
- Resolution: Scale based on map size (max 2048x2048)
- Quality: 70-80% JPEG compression
- Only capture when viewers have adopted colonists

**Option B: Cell Data Streaming** (Alternative)
Send structured map data instead of images:

```csharp
public MapDataSnapshot CaptureMapData()
```

### 5.2: Frontend - Interactive Map Renderer

**Canvas-based renderer**:

```javascript
// MapRenderer.js
class InteractiveMap {
  constructor(canvasId) {
    this.canvas = document.getElementById(canvasId);
    this.ctx = this.canvas.getContext('2d');
    this.mapImage = new Image();
    this.zoom = 1.0;
    this.panX = 0;
    this.panY = 0;
  }

  updateMap(imageData) {
    this.mapImage.src = 'data:image/jpeg;base64,' + imageData;
    this.render();
  }

  render() {
    // Draw map with zoom/pan
    this.ctx.clearRect(0, 0, this.canvas.width, this.canvas.height);
    this.ctx.save();
    this.ctx.translate(this.panX, this.panY);
    this.ctx.scale(this.zoom, this.zoom);
    this.ctx.drawImage(this.mapImage, 0, 0);
    this.ctx.restore();

    // Draw colonist marker
    this.drawColonistMarker();
  }

  onClick(event) {
    // Convert screen coords to map coords
    const mapPos = this.screenToMap(event.x, event.y);

    // Send move command
    sendColonistCommand({
      action: 'move',
      target: { x: mapPos.x, z: mapPos.z }
    });
  }
}
```

**Features**:

## CLOUDFLARE STREAM INTEGRATION (Optional / Future)
================================================================================
**Goal**: Replace WebRTC/HLS setup with Cloudflare Stream for enterprise-grade
low-latency live streaming (LL-HLS) and simpler infrastructure.

### Implementation Overview:
1.  **Mod**: Capture frames via FFmpeg sidecar (already done) but push to RTMPS.
2.  **Sidecar**: Update `sidecar.go` to support RTMPS output to Cloudflare ingest.
3.  **Cloudflare**: Create Live Input via API.
4.  **Frontend**: Use Cloudflare Stream Player.

### Steps:
1.  **API Integration**:
    *   Create endpoint `POST /api/stream/create`
    *   Call Cloudflare API to create Live Input
    *   Return `rtmps_url` and `stream_key` to Mod
2.  **Mod Update**:
    *   Receive RTMPS credentials
    *   Pass to `sidecar.exe`
3.  **Sidecar Update**:
    *   FFmpeg command: `ffmpeg -f rawvideo ... -c:v libx264 -f flv rtmps://...`
4.  **Frontend**:
    *   Embed `<stream src="...">` or use HLS.js with Cloudflare HLS URL.

### Pros:
*   Global CDN distribution (no manual BunnyCDN setup)
*   Automatic recording/VODs
*   Adaptive bitrate streaming (ABR)
*   Sub-second latency with LL-HLS

### Cons:
*   Cost ($5/1000 mins viewed) - might be expensive for 24/7 streams
*   Requires Cloudflare account and API keys
